Напиши Python-класс TokenBucket — ограничитель скорости методом
"токенного ведра" (типовая задача: троттлинг запросов к API).

TokenBucket(capacity: float, refill_rate: float)
  capacity — максимум токенов в ведре.
  refill_rate — сколько токенов добавляется в секунду.
  Ведро стартует ПОЛНЫМ.

Методы:

1. consume(tokens: float, now: float) -> bool
   Сначала пополняет ведро по времени, прошедшему с прошлого
   обращения (но не выше capacity), затем пытается списать tokens.
   Если токенов хватает — списывает и возвращает True, иначе
   НЕ списывает и возвращает False.

2. available(now: float) -> float
   Текущее число токенов с учётом пополнения на момент now
   (не больше capacity), без списания.

Тонкости:
- now монотонно неубывает в сценарии; первый вызов задаёт точку отсчёта.
  Считай, что внутренний "последний момент" инициализируется при
  первом вызове любого метода значением now этого вызова.
- Пополнение непрерывное: за dt секунд добавляется dt*refill_rate.
- Запрос tokens > capacity никогда не может быть удовлетворён -> False
  (и не списывает).
- consume с tokens <= 0 -> ValueError. capacity <= 0 или
  refill_rate < 0 -> ValueError в конструкторе.
- Сравнение токенов с допуском 1e-9 (из-за float): если не хватает
  буквально на эпсилон — считать что хватает.
