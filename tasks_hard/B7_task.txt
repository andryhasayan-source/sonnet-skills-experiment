Напиши модуль для Telegram-бота на aiogram 3.x: парсер команды с
аргументами и флагами (типовая задача: гибкий разбор пользовательского
ввода вида "/remind 15m купить хлеб --silent").

ВАЖНО: только модуль, запуск polling — под if __name__ == "__main__".

На уровне модуля:

1. def parse_duration(token: str) -> int | None
   "15m"->900, "2h"->7200, "30s"->30, "1d"->86400 (в секундах).
   Допустимы суффиксы s/m/h/d, число — положительное целое.
   Иначе None. "0m" -> None (положительное). "1.5h" -> None.

2. def parse_command(text: str) -> dict | None
   Разбирает "/cmd arg1 arg2 ... --flag1 --flag2".
   Возвращает {"command": str (без слэша),
               "args": [позиционные токены по порядку],
               "flags": set(имена флагов без --)}.
   Правила:
   - первый токен обязан начинаться с "/" и иметь хотя бы 1 символ
     после слэша, иначе None;
   - флаг — токен, начинающийся с "--" и имеющий имя после (>=1 символ),
     состоящее из [a-z0-9_]; "--" без имени или с другими символами -> None;
   - флаги могут идти вперемешку с аргументами, но в args попадают
     только НЕ-флаги;
   - повторяющиеся флаги схлопываются (set);
   - лишние пробелы между токенами игнорируются;
   - пустая строка или только пробелы -> None.

3. def build_reminder(text: str) -> dict | None
   Для команды remind: первый позиционный arg — длительность
   (parse_duration), остальные args — текст напоминания (склеить
   пробелом). Возвращает {"seconds": int, "message": str,
   "silent": bool (есть ли флаг silent)} или None, если команда не
   "remind", нет валидной длительности или пустой текст.
