Напиши модуль для Telegram-бота на aiogram 3.x: корзина интернет-магазина
с количеством товаров (типовая, но с тонкостями состояния).

ВАЖНО: только модуль, запуск polling — под if __name__ == "__main__".
Логику храни в тестируемом классе, обработчики — тонкие обёртки.

На уровне модуля:

1. class Cart:
   - __init__(self)
   - add(self, user_id, product, qty=1) -> int — добавляет qty штук
     товара, возвращает НОВОЕ суммарное количество этого товара
     у пользователя. qty <= 0 -> ValueError.
   - remove(self, user_id, product, qty=1) -> int — убирает qty штук;
     если количество падает до 0 или ниже — товар полностью удаляется
     из корзины; возвращает новое количество (0 если удалён).
     qty <= 0 -> ValueError.
   - items(self, user_id) -> dict — копия {товар: количество}
     (изменение возвращённого словаря НЕ влияет на корзину).
   - total_qty(self, user_id) -> int — сумма всех количеств.
   - clear(self, user_id) -> None
   - корзины пользователей независимы.

2. router = Router()

3. PRICES = {"apple": 50, "bread": 40, "milk": 80}

4. def cart_summary(cart: Cart, user_id: int) -> str
   Если корзина пуста -> "Корзина пуста.".
   Иначе строки вида "apple x3 = 150 руб." по каждому товару
   (в порядке добавления — учти это в Cart!), и последней строкой
   "Итого: N руб." Товары не из PRICES в сумму идут по цене 0.

Ответы обработчиков — через await message.answer(...).
