Напиши модуль для Telegram-бота на aiogram 3.x: брокер-распределитель
заявок между операторами с балансировкой (логика в тестируемом классе).

ВАЖНО: только модуль, запуск polling — под if __name__ == "__main__".

На уровне модуля:

1. class Dispatcher:
   - __init__(self)
   - add_operator(self, op_id: int) -> None — добавляет оператора
     (повторное добавление того же — без эффекта, не дублировать).
   - remove_operator(self, op_id: int) -> list — убирает оператора;
     все его НЕзакрытые заявки возвращаются и должны быть
     перераспределены вызывающим (метод возвращает список id заявок,
     которые были на этом операторе). Если оператора нет — [].
   - assign(self, ticket_id: int) -> int | None — назначает заявку
     оператору с наименьшим числом активных заявок (при равенстве —
     тому, кто был добавлен раньше). Возвращает op_id или None,
     если операторов нет. Повторный assign того же ticket_id, если он
     уже активен, -> возвращает текущего оператора, НЕ переназначая.
   - close(self, ticket_id: int) -> bool — закрывает заявку
     (снимает с оператора), True если была активна, иначе False.
   - load(self, op_id: int) -> int — число активных заявок оператора.

2. router = Router()  (без обработчиков, для единообразия)

Балансировку и стабильный порядок при равенстве нагрузки —
реализуй аккуратно, это главная сложность.
