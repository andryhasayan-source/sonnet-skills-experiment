Напиши модуль для Telegram-бота на aiogram 3.x: многошаговый опрос (FSM)
с возможностью вернуться назад командой /back.

ВАЖНО: только модуль, запуск polling — под if __name__ == "__main__".

Сценарий /survey: вопрос1 "Ваш возраст?" -> вопрос2 "Ваш город?"
-> вопрос3 "Ваш email?" -> итог.

На уровне модуля:

1. router = Router()
2. class Survey(StatesGroup): age, city, email
3. QUESTIONS = {"age": "Ваш возраст?", "city": "Ваш город?", "email": "Ваш email?"}
4. ORDER = ["age", "city", "email"]

5. def validate(field: str, value: str) -> str | None
   Возвращает None если значение валидно, иначе текст ошибки:
   - age: целое 1..120, иначе "Возраст должен быть числом от 1 до 120."
   - city: непустая строка >= 2 символов после strip,
     иначе "Город слишком короткий."
   - email: содержит ровно один "@", непустые части до и после,
     в доменной части есть ".", иначе "Некорректный email."

6. async def cmd_survey(message, state) — ставит Survey.age,
   спрашивает QUESTIONS["age"], сохраняет в state служебное поле
   с именем текущего шага.

7. async def cmd_back(message, state) — команда /back:
   если текущий шаг первый (age) — "Вы на первом вопросе." и остаёмся;
   иначе переходим на ПРЕДЫДУЩИЙ шаг, повторяем его вопрос.
   Должно работать из любого шага.

8. По одному обработчику на шаг (process_age/process_city/process_email)
   ИЛИ один обобщённый — на ваш выбор, но поведение:
   - если /back — обрабатывается cmd_back (учти приоритет);
   - если validate вернул ошибку — показать её, остаться на шаге;
   - если ок — сохранить значение, перейти к следующему шагу и задать
     его вопрос; после email — показать
     "Готово! Возраст: {age}, город: {city}, email: {email}" и state.clear().

Подсказка по тестам: тесты будут вызывать process_*-обработчики и
cmd_back напрямую с мок-объектами message и state, проверяя
вызовы state.set_state / update_data / get_data и тексты ответов.
Поэтому храни в state данные под ключами "age","city","email".
