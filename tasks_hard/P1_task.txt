Напиши Python-модуль для работы с римскими числами со СТРОГОЙ валидацией.

Функции:

1. int_to_roman(n: int) -> str
   Переводит 1..3999 в каноническую римскую запись.
   Для n вне 1..3999 или не-int — поднять ValueError.

2. roman_to_int(s: str) -> int
   Переводит корректную римскую запись в число.
   ВАЖНО: принимаются ТОЛЬКО канонические записи. Если запись
   некорректна — поднять ValueError. Некорректны, например:
   - "IIII" (4 одинаковых подряд для I/X/C/M; и любые 2+ подряд для V/L/D)
   - "VV", "LL", "DD" (V, L, D не повторяются)
   - "IC", "IL", "XM", "VX" (недопустимые вычитания: вычитать можно
     только I из V/X, X из L/C, C из D/M; одна вычитаемая цифра)
   - "IXI", "IVIV" (после вычитающей пары нельзя ставить цифру того же
     или большего разряда, дающую неканоничность)
   - пустая строка, строчные буквы, посторонние символы
   Проще говоря: roman_to_int(s) корректна тогда и только тогда, когда
   int_to_roman(roman_to_int(s)) == s. Реализуй так, чтобы это держалось.

Регистр строго верхний. На не-str входе — ValueError.
Выдай один файл кода.
