Напиши Python-компаратор семантических версий (SemVer 2.0,
типовая задача для менеджеров зависимостей).

Функции:

1. parse_version(s: str) -> tuple | None
   Разбирает строку вида MAJOR.MINOR.PATCH с опциональным
   пред-релизом "-alpha.1" и метаданными сборки "+build.5".
   Пример: "1.2.3-alpha.1+build.7".
   Возвращает структуру, достаточную для сравнения, или None
   если версия невалидна (отрицательные числа, ведущие нули в
   числовых частях типа "01", лишние точки, пустые сегменты).

2. compare(a: str, b: str) -> int
   -1 если a < b, 0 если равны, 1 если a > b.
   Если хотя бы одна не парсится — ValueError.
   Правила сравнения по SemVer 2.0:
   - сравниваем MAJOR, затем MINOR, затем PATCH численно;
   - версия С пред-релизом МЕНЬШЕ версии без него при равных
     MAJOR.MINOR.PATCH (1.0.0-alpha < 1.0.0);
   - пред-релизные идентификаторы сравниваются по точкам:
     числовые — численно, нечисловые — лексикографически,
     числовой идентификатор всегда меньше нечислового,
     больший набор идентификаторов больше при совпадающем префиксе
     (1.0.0-alpha < 1.0.0-alpha.1);
   - метаданные сборки (+...) НЕ влияют на сравнение
     (1.0.0+a == 1.0.0+b -> 0).

Выдай один файл кода.
