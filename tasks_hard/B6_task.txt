Напиши модуль для Telegram-бота на aiogram 3.x: викторина с подсчётом
очков и таймаутом ответа (логика — в тестируемом классе).

ВАЖНО: только модуль, запуск polling — под if __name__ == "__main__".

На уровне модуля:

1. QUESTIONS = [
     {"q": "Столица Франции?", "a": "париж"},
     {"q": "2+2?", "a": "4"},
     {"q": "Цвет неба?", "a": "голубой"},
   ]

2. class Quiz:
   - __init__(self, questions=QUESTIONS, time_limit=30.0)
   - start(self, user_id, now) -> str — начинает викторину для игрока
     (сбрасывает прогресс/очки), возвращает текст первого вопроса.
   - answer(self, user_id, text, now) -> dict — обрабатывает ответ.
     Возвращает dict с ключами:
       "correct": bool — верен ли ответ (сравнение без регистра и
                  с обрезкой пробелов),
       "timed_out": bool — был ли превышен time_limit с момента
                  выдачи текущего вопроса (now - время_выдачи > time_limit);
                  при таймауте ответ НЕ засчитывается как правильный,
                  даже если текст совпал,
       "finished": bool — закончились ли вопросы,
       "next_question": str | None — текст следующего вопроса или None,
       "score": int — текущее число правильных.
     Если игра не начата или уже закончена — вернуть dict с
     "finished": True, "next_question": None, остальные поля разумно.
     Каждый ответ (верный, неверный или таймаут) продвигает к
     следующему вопросу. Время выдачи следующего вопроса = now
     текущего ответа.
   - score(self, user_id) -> int

Players независимы. Это основная сложность — корректно вести
время выдачи вопроса и продвижение.
