Напиши Python-функцию stable_topk для выбора top-K элементов
(типовая задача: рейтинги, лидерборды).

stable_topk(items: list, k: int, key=None) -> list

Возвращает k элементов с наибольшим ключом, в порядке убывания ключа.
Тонкости:
1. При равных ключах сохраняется ИСХОДНЫЙ относительный порядок
   (стабильность): среди равных раньше идёт тот, кто был раньше в items.
2. key — функция извлечения ключа; если None, сравниваются сами элементы.
3. k <= 0 -> пустой список. k больше len(items) -> вернуть все
   (отсортированные по убыванию ключа со стабильностью).
4. items не мутируется.
5. Если ключи несравнимы между собой (разные типы, TypeError при
   сравнении) — пробросить TypeError наружу, не глотать.
6. Сложность — лучше чем полная сортировка не требуется, но результат
   обязан быть корректным; можно сортировать.

Примеры:
stable_topk([3,1,2,3,1], 2) -> [3,3]
stable_topk([("a",2),("b",1),("c",2)], 2, key=lambda x:x[1])
   -> [("a",2),("c",2)]
